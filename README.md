# My CISSP Journey

> Not just studying for an exam — evolving how I think about security.

---

## 👋 Who I Am

I’m a Senior Network Security Engineer who has spent years in the trenches:

- Upgrading firewalls  
- Cleaning up policy sprawl  
- Solving VPN compliance issues  
- Supporting cloud migrations  
- Hardening enterprise environments  

I’ve built, fixed, optimized, and protected networks.

Now I’m leveling up.

This repository documents my journey toward earning the CISSP — and more importantly, shifting from **technical executor** to **enterprise security leader**.

---

## 🎯 Why I’m Pursuing CISSP

For me, CISSP isn’t about adding letters to my name.

It’s about:

- Thinking in risk, not just controls  
- Designing security programs, not just firewall rules  
- Understanding governance and business impact  
- Strengthening leadership credibility  

I want to operate at the level where security strategy is shaped — not just implemented.

---

## 📅 The Plan

I’m following a structured 90-day sprint:

- One domain at a time  
- Deep conceptual understanding  
- Practice question breakdowns  
- Weekly reflections  
- Real-world mapping to my current engineering work  

This repo is my accountability system.

---

## 🧠 What You’ll Find Here

Each domain folder includes:

- Clear notes in my own words  
- Executive-level explanations  
- Missed question analysis  
- Diagrams and breakdowns  
- “Engineer → Leadership” mindset shifts  

I’m not just memorizing definitions.

I’m training how I think.

## 🔄 Real-World Application

One thing I’m doing differently:

I’m mapping CISSP concepts to actual enterprise work I’ve done — like:

- Firewall lifecycle upgrades  
- IAM enforcement decisions  
- VPN compliance strategy  
- Risk-based policy cleanups  
- Cloud integration security considerations  

Security theory is powerful.  
Security applied in production is mastery.

---

## 🧩 The Mindset Shift

One of the biggest lessons so far:

| Old Thinking | New Thinking |
|--------------|-------------|
| Fix the vulnerability | Assess the business risk |
| Deploy the control | Evaluate control effectiveness |
| Solve the ticket | Strengthen the security program |
| Technical depth | Strategic impact |

CISSP forces you to zoom out.

And I’m leaning into that.
